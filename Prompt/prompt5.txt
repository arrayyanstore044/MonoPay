Isi dari planningnya adalah sebagaia berikut :

Buatkan API untuk logout user

Endpoint : DELETE /api/users/logout

Headers :
- Authorization : Bearer <token> (token adalah token yang ada di table users)

Response Body (Success) :
 {
    "status" : "OK!"
    "message": "User Log Out Successfully!"
    "data": {
        "user": {
            "id": 1,
            "name": "Jhon",
            "email": "[EMAIL_ADDRESS]",
            "created_at": "timestamp"
        }
    }
 }

Jika sukses logout, maka data session dengan token tersebut harus di hapus dari table user_session

 Response Body (Failed) :
 { 
    "status": "ERROR",
    "message": "UnAuthorized.",
    "data": null
}
